# 影灯设计师 MVP

影灯设计师是一个用于生成投影灯盒的本地 Web 工具。用户上传一张适合做剪影的图片后，应用会把图片转成二值投影目标，计算矩形贴墙灯盒四个侧壁应该保留或镂空的结构，并导出可用于 3D 打印的 STL 模型。

当前项目仍是 MVP，目标不是完美还原任意图片，而是验证一条可行链路：图片输入 -> 投影拆解 -> 可打印结构生成 -> 3D 预览 -> STL 导出。

## 项目目的

这个项目想解决的问题是：普通用户如何把一张高对比度图片快速转成一个可打印、可装灯、贴墙后能投出近似轮廓的实体灯盒。

核心设计约束：

- 灯盒是矩形贴墙结构，背面贴墙，正面开放。
- 背板默认是实体，会遮挡墙面中心区域，这是物理结构决定的结果。
- 目标投影主要出现在灯盒四周展开区域，由左、右、上、下四个侧壁的镂空共同形成。
- 模型必须优先考虑可打印性，不能为了保留图片细节生成大量悬浮碎片或过细连接。
- 所有图片处理、几何计算和 STL 导出默认在浏览器本地完成，不依赖后端上传图片。

## 当前能力

当前版本已经支持：

- 上传 JPG / PNG / WEBP 图片，或使用内置示例图。
- 基于纯色背景的前景提取、反色、背景灵敏度调节。
- 设置灯盒宽度、高度、深度、壁厚、最小特征尺寸、采样精度。
- 设置墙面投影倍率、光源距墙高度、光源亮度。
- 自动把目标投影拆分到四个侧壁 mask。
- 对模型做全局连通性检查，尽量去除悬浮碎片。
- 对长细条结构做剪枝，减少不可打印或视觉上像虚线的连接。
- 使用 Three.js 做 3D 预览和投影效果预览。
- 导出 STL，包含灯盒主体、背板和 3mm 直径灯柱。
- 提供普通启动脚本和强兼容 WebGL 启动脚本。

## 快速开始

### 方式一：一键启动

Windows 下直接双击：

- `一键启动完整服务.cmd`
- 如果浏览器 WebGL 不稳定，使用 `一键启动完整服务(强兼容WebGL).cmd`

脚本会自动检查依赖、启动 Vite 服务，并打开浏览器。

### 方式二：命令行启动

```bash
npm install
npm run dev
```

默认访问：

```text
http://127.0.0.1:5173/
```

### 构建

```bash
npm run build
```

构建产物在 `dist/`。项目里有 `scripts/postbuild-file-open-fix.mjs`，用于让构建后的 HTML 在部分本地打开场景下更容易加载样式和脚本。但推荐优先通过本地服务访问，因为 3D 预览依赖浏览器的 WebGL 能力。

## 当前架构

项目是一个纯前端单页应用：

```text
index.html
  -> src/main.ts        应用入口、状态、图片处理、投影计算、几何生成、STL 导出
  -> src/styles.css     工具型 UI 样式
scripts/
  -> postbuild-file-open-fix.mjs
start-full-service.ps1  Windows 一键启动服务脚本
```

主要技术栈：

- Vite：开发服务器和构建工具。
- TypeScript：主逻辑实现语言。
- Three.js：3D 预览、灯光、相机控制、STL 导出。
- Canvas 2D API：图片读取、背景检测、二值 mask 生成。
- 自定义网格生成：根据侧壁 mask 生成可打印几何。

当前没有后端服务，没有数据库，也没有云端图片处理。

## 核心流程

```text
上传图片
  -> Canvas 读取像素
  -> 背景颜色估计
  -> 前景/背景二值化
  -> 轻度形态学清理和小碎片过滤
  -> 生成目标投影 mask
  -> 从光源反向追踪到墙面目标点
  -> 计算光线命中的侧壁
  -> 写入左/右/上/下四个侧壁 mask
  -> 根据最小特征和边框参数清理 mask
  -> 全局连通性检测
  -> 删除过小悬浮碎片和长细条结构
  -> 必要时补连接
  -> 生成 3D 网格
  -> Three.js 预览
  -> STL 导出
```

## 几何建模策略

早期版本按很多小盒子拼接墙体，连续区域会出现大量内部切片面，视觉上容易像一排排细条。当前版本改为按 mask 生成外露表面：

- 连续实体区域先做矩形合并。
- 外侧面和内侧面只生成可见大面。
- 孔洞边界和实体边界生成厚度侧面。
- 内部相邻单元之间不再生成重复侧面。

这样可以减少 STL 中的条纹感，并降低无意义面片数量。

## 可打印性处理

当前 MVP 使用几类保守规则提高可打印性：

- 最小特征尺寸限制。
- 小连通域过滤。
- 全局连通性 flood fill。
- 必要连接桥补全。
- 长细条结构剪枝。
- 背面结构保护，避免模型整体脱离背板。

这仍然不是工业级网格修复。复杂图片可能仍需要用户调高最小特征尺寸、降低采样精度或换更清晰的剪影图。

## 已知限制

- 只支持矩形贴墙灯盒。
- 不保证任意图片都能生成可打印模型。
- 投影效果是近似预览，真实效果会受光源尺寸、墙距、材料厚度、打印误差影响。
- 当前没有真正的流形网格布尔运算和专业网格修复。
- 当前 UI 和算法主要为单用户本地使用设计，还不是产品化服务。
- `PRD.md` 当前存在编码显示问题，后续应整理为 UTF-8 文档。

## 希望实现的最终形态

最终形态希望成为一个面向普通用户和创客的「投影灯盒设计器」：

### 1. 更稳定的图片到结构转换

- 支持透明背景、纯色背景、复杂背景抠图。
- 支持自动简化轮廓、保留关键视觉特征。
- 支持用户手动擦除、补线、加粗、连接关键区域。
- 支持投影效果评分，提示哪些区域无法投出或不可打印。

### 2. 更可靠的可打印模型生成

- 引入真正的轮廓矢量化和多边形布尔运算。
- 使用 Manifold / CSG / 网格修复库生成水密模型。
- 导出前做非流形边、孤岛、薄壁、悬空结构检测。
- 提供不同打印机和喷嘴尺寸下的预设规则。

### 3. 更真实的光学预览

- 区分点光源、LED 珠、漫射光源等光源类型。
- 支持墙距、灯源位置、灯源尺寸、材料厚度对投影的影响。
- 显示投影清晰度、遮挡区、亮度衰减。
- 支持多视角对比：模型视图、墙面投影视图、侧壁展开图。

### 4. 更完整的产品结构

- 支持安装孔、走线孔、灯座、卡扣、背板固定结构。
- 支持导出 STL / 3MF / OBJ / GLB。
- 支持导出四个侧壁的 2D 展开 mask。
- 支持保存项目、复用参数、生成打印说明。

### 5. 更丰富的形态

- 矩形贴墙盒继续作为第一优先级。
- 后续可扩展到正六边形、多边形、圆柱、异形外壳。
- 每种形态都需要重新定义光线追踪、侧壁展开和连通结构规则，不能只改外观。

## 推荐输入图片

为了得到更好的结果，建议使用：

- 纯色背景。
- 主体和背景对比强。
- 剪影清晰，少渐变，少半透明。
- 主体结构尽量连通。
- 不要包含过多非常细的发丝、纹理、网格、碎片。

## 当前仓库状态

这是一个本地优先的实验性 MVP。它已经可以完成端到端流程，但算法仍在迭代，尤其是“投影效果”和“可打印结构”之间的平衡。

短期优先级：

1. 继续减少不必要的细条连接。
2. 提升边缘清洁度和细节保留能力。
3. 增加导出前的模型质量检查。
4. 整理并修复 `PRD.md` 的编码问题。
