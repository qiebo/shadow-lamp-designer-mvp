## 1. 产品概述

### 1.1 一句话描述

一个纯前端 Web 应用。用户上传一张图片后，系统将图片处理为可制造的镂空图案，并生成一个**矩形截面、背面贴墙、正面开放**的 3D 打印灯盒模型。灯盒内部放置点光源后，光线穿过四个侧壁的镂空区域，在墙面上形成接近预设图片外轮廓的投影效果。

### 1.2 关键设计修正

本版本明确采用**矩形贴墙结构**：

- 灯盒背面贴墙，默认是实心背板。
- 实心背板会遮挡投影中心区域，这是正常物理结果，不作为缺陷处理。
- 产品目标不是让墙上每个像素都完整还原原图，而是让墙面可见投影区域的**整体轮廓、姿态和识别度**接近预设图片。
- 图案会被拆分到上、下、左、右四个侧壁；四个侧壁的光斑共同组成墙面上的目标轮廓。
- 初版只支持矩形盒体，不支持圆形、椭圆、多边形或自定义轮廓。

### 1.3 目标用户

- 有 3D 打印机的创客/手工艺爱好者
- 想要定制个性化投影夜灯的普通用户
- 需要快速生成 3D 打印灯盒方案的创意设计师

### 1.4 核心价值

- **快速生成**：上传图片后生成矩形投影灯盒模型。
- **物理边界清晰**：承认背板遮挡，不承诺无遮挡完整投影。
- **打印优先**：通过去噪、连通域过滤、最小特征尺寸限制，提高 STL 可打印性。
- **纯前端隐私**：图片处理和模型生成默认在浏览器本地完成。

## 2. 产品原理

### 2.1 物理模型

```
墙面 z=0
┌──────────────────────────────────────┐
│     可见投影区域围绕灯盒展开          │
│                                      │
│        ┌──────────────────┐          │
│        │  实心背板遮挡区  │          │
│        │  灯盒贴墙位置    │          │
│        └──────────────────┘          │
│                                      │
└──────────────────────────────────────┘

侧视结构

墙面 / 背板 z=0                正面开口 z=BD
│                                     │
│  ┌────────────────────────────────┐ │
│  │           顶壁镂空              │ │
│  │                                │ │
│  │  左壁镂空    点光源    右壁镂空 │ │
│  │                                │ │
│  │           底壁镂空              │ │
│  └────────────────────────────────┘ │
│  实心背板贴墙，默认遮挡中心投影       │
```

### 2.2 盒体结构

| 部件 | 位置 | 说明 |
|---|---|---|
| 背板 | `z=0` | 实心，贴墙，用于固定和结构支撑；会遮挡中心投影 |
| 正面 | `z=BD` | 开放，无面，方便安装/更换灯源 |
| 左壁 | `x=-BW/2` | 镂空壁板，参与投影 |
| 右壁 | `x=BW/2` | 镂空壁板，参与投影 |
| 顶壁 | `y=BH/2` | 镂空壁板，参与投影 |
| 底壁 | `y=-BH/2` | 镂空壁板，参与投影 |
| 光源 | 盒体内部 | 默认位于盒体中心附近，可沿 Z 轴前后调节 |

### 2.3 投影目标

本产品的投影目标定义为：

1. 用户上传或选择一张目标图片。
2. 系统提取图片主体，生成目标投影轮廓。
3. 根据矩形灯盒、点光源和墙面的几何关系，将目标轮廓拆分到四个侧壁。
4. 四个侧壁镂空后的出光区域投射到墙上，形成一个围绕灯盒展开的可见投影。
5. 背板遮挡区内的目标内容默认不可见；系统可在预览中显示遮挡区，帮助用户理解最终效果。

### 2.4 投影几何

坐标约定：

```text
Y ↑
  |
  +----→ X
  |
Z 远离墙面

墙面位于 XY 平面，z=0
背板位于 z=0
盒体沿 +Z 方向延伸
正面开口位于 z=BD
```

反向光线追踪流程：

1. 对目标图片中的前景像素，映射到墙面目标点 `P_wall = (x, y, 0)`。
2. 从光源 `P_light = (0, 0, z_light)` 指向墙面点构造射线。
3. 计算射线与四个侧壁平面的交点。
4. 取位于盒体深度范围 `0 <= z <= BD` 内、且最先命中的侧壁交点。
5. 将交点投影到对应壁板的 2D 坐标，标记为该壁板的镂空区域。
6. 若射线被背板遮挡或没有命中有效侧壁，则该像素无法投影，标记为遮挡/丢弃。

核心认知：

- 四个侧壁共同生成投影轮廓。
- 背板对应的中心遮挡区域不参与投影。
- 系统追求视觉上可识别的轮廓，而不是严格逐像素还原。

## 3. 功能需求

## 4. 技术方案

### 4.1 推荐技术栈

开发阶段建议使用模块化工程，不建议直接从单文件 HTML 起步。

| 模块 | 推荐技术 | 说明 |
|------|---------|------|
| 前端工程 | Vite + TypeScript | 便于维护复杂几何和状态 |
| UI | 原生 CSS 或轻量组件 | 深色工具型界面，不做营销页 |
| 3D 渲染 | Three.js | 预览、灯光、相机、交互 |
| 射线计算 | three-mesh-bvh / 自定义平面求交 | 矩形盒体可先用自定义平面求交 |
| 图片处理 | Canvas 2D API | P0 足够；P1 可引入 OpenCV.js |
| 轮廓处理 | marching squares / potrace / clipper | 将 bitmap 转为可制造轮廓 |
| 实体建模 | Manifold WASM 优先 | 更适合生成可打印网格 |
| 备选 CSG | three-bvh-csg | 可用于预览或简单布尔，但不作为唯一可靠导出方案 |
| 导出 | STLExporter / 自定义 STL writer | 输出 STL，单位按 mm |
| 测试 | Playwright | 检查上传、参数调整、预览和导出流程 |

### 4.2 MVP 可接受方案

为了快速验证产品，可先采用：

- 单页面 Vite 应用。
- Three.js 做 3D 预览。
- Canvas 2D 做图片分割和 mask。
- 矩形平面求交做反向光线追踪。
- 壁面 mask 先用网格化方式生成孔洞。
- STL 导出先支持中低复杂度图片。

MVP 不承诺：

- 不支持非矩形截面。
- 不保证任意复杂图片都能稳定打印。
- 不保证投影逐像素还原。
- 不保证背板遮挡区有投影内容。

### 4.3 核心算法流程

```text
用户上传图片
  ↓
Canvas 2D 背景检测和二值化
  ↓
去噪、连通域过滤、最小特征处理
  ↓
生成目标投影 mask
  ↓
反向光线追踪：
  对目标 mask 采样点
  从光源连到墙面点
  判断是否命中四个侧壁
  过滤背板遮挡/无效区域
  写入对应壁面 mask
  ↓
壁面 mask 转换为镂空几何
  ↓
生成矩形灯盒：
  实心背板 + 四个镂空侧壁 + 正面开放
  ↓
Three.js 预览实体和投影效果
  ↓
导出 STL
```

### 4.4 关键实现策略

#### 图片到可制造轮廓

- 不直接按每个像素执行一次 CSG。
- 先把图片降低到合理分辨率。
- 对 mask 做去噪、闭合、连通域过滤。
- 将 bitmap 转换为简化轮廓。
- 对过细区域进行删除、加粗或提示。

#### 墙面遮挡区处理

- 背板尺寸对应墙面上的中心遮挡区域。
- 预览中使用半透明矩形标记遮挡区。
- 遮挡区内的目标像素不参与最终可见投影评分。
- 导出前提示用户：该结构会遮挡图片中心区域。

#### 模型可打印性

- 壁厚默认不低于 1.2mm。
- 镂空边缘距离外框保留安全边距。
- 小孔、小桥、孤岛需要过滤或加粗。
- 对复杂图片提供“简化强度”参数。

## 6. 3D 打印约束

| 参数 | 约束 | 说明 |
|------|------|------|
| 最小壁厚 | >= 1.2mm | 太薄容易断裂 |
| 最小特征尺寸 | >= 0.8mm | 小于此值可能打印失败 |
| 建议特征尺寸 | >= 1.2mm | 更适合普通 FDM 打印 |
| 填充率 | 20-30% | 灯盒不需要完全实心 |
| 打印方向 | 需根据模型提示选择 | 尽量减少支撑 |
| 模型连通性 | 必须尽量保证 | 孤岛和悬浮碎片需过滤 |

## 7. 已知风险

| 风险 | 等级 | 说明 | 应对 |
|------|------|------|------|
| 背板遮挡导致中心缺失 | 高 | 物理结构决定 | 在预览中明确显示遮挡区 |
| 复杂图片导致模型过重 | 高 | 孔洞太多、轮廓太复杂 | 分辨率限制、轮廓简化 |
| 小碎片不可打印 | 高 | 图片转镂空天然容易产生孤岛 | 连通域过滤、最小特征检查 |
| 浏览器 CSG 性能不稳定 | 中 | 大规模布尔运算可能卡顿 | 优先轮廓建模，减少 CSG |
| 阴影预览不等于真实灯效 | 中 | 真实光源、材料、墙距会影响效果 | 标注为近似预览 |

## 9. 验收标准

1. 上传一张高对比度 PNG 后，系统能生成清晰二值轮廓。
2. 调整盒体宽、高、深、光源位置后，四侧壁 mask 会重新计算。
3. 3D 预览中能看到实心背板、正面开口和四个镂空侧壁。
4. 墙面预览中能看到中心遮挡区，并能看到围绕遮挡区展开的目标轮廓投影。
5. 导出的 STL 单位为毫米，尺寸与参数一致。
6. 对过细、过碎、不可投影比例过高的图片，系统能给出提示。
